# Cookie 同意弹窗项目说明

## 1. 项目目标

本项目用于练习 JavaScript 的基础 DOM 操作和事件处理能力，模拟真实网站中的 Cookie 同意弹窗场景。

用户首次访问页面时，右下角会出现弹窗，告知站点使用 Cookie。用户点击同意后，弹窗消失并记录同意状态，后续访问不再重复出现。

## 2. 已实现功能

- 首次访问显示 Cookie 同意弹窗
- 点击按钮后弹窗关闭
- 使用 localStorage 持久化用户同意状态
- 已同意用户刷新或再次访问时不再弹出
- 提供关闭按钮（仅关闭，不记录同意）
- 弹窗带过渡动画，页面支持移动端显示

## 3. 项目结构

- index.html：页面结构、弹窗结构、按钮与容器
- style.css：页面布局、弹窗样式、动画与响应式适配
- script.js：DOM 选择、事件绑定、状态读写与显示控制
- README.md：简要说明
- PROJECT_DESCRIPTION.md：完整项目说明（本文件）

## 4. 运行方式

1. 打开项目目录下的 index.html。
2. 首次打开时可以看到右下角 Cookie 弹窗。
3. 点击 I like Cookies 后刷新页面，弹窗不再出现。

## 5. 核心技术点

### 5.1 DOM 操作

通过 getElementById 获取关键节点：

- cookiePopup：弹窗容器
- acceptCookieBtn：同意按钮
- closePopupBtn：关闭按钮

### 5.2 事件处理

使用 addEventListener 监听按钮点击：

- 同意按钮：保存同意状态并关闭弹窗
- 关闭按钮：仅关闭弹窗

### 5.3 状态持久化

- 存储键名：cookieConsentAccepted
- 同意值：yes
- 存储方式：localStorage

判断逻辑：

- 如果 localStorage 中不存在 yes，页面加载后延迟约 300ms 显示弹窗
- 如果已存在 yes，则不显示弹窗

## 6. 页面与交互设计说明

- 弹窗固定在右下角，符合常见 Cookie 通知位置
- 使用深色弹窗与浅色按钮提高视觉层级
- 通过 show 类切换控制显隐状态
- 显示时使用透明度与位移动画增强体验
- 移动端缩小边距，保证弹窗可读性与点击区域

## 7. 与题目要求的对应关系

题目要求：创建 Cookie 同意弹窗，包含提示消息和同意按钮，点击后消失。

本项目完成情况：
